What is JSX?

It's an XML-like syntax to describe components.

It is embedded into JavaScript code.

It's syntactic sugar for React primitives.

# JSX at a glance ```js // Declaration of a class var Avatar = React.createClass({ render: function() { return ( <div> <img src={'https://graph.facebook.com/' + this.props.userid + '/picture'} /> </div> ); } }); // Call to render a component ReactDOM.render(<Avatar userid="claudiopro" />, document.querySelector('#mount-point')); ```
# Gotchas - React has a whitelist for HTML element names and attributes. If you pass elements or attribute names that are not in the list, React will not render them. - You can pass JavaScript strings as content or attribute values. Note that React performs escaping on dynamic content.

Simple transform

React JSX transforms from XML-like syntax into JavaScript. XML elements, attributes and children are transformed into arguments passed to React.createElement.


var Avatar = require('./Avatar');

// Input (JSX):
var app = <Avatar userid="claudiopro" />;


// Output (JS):
var app = React.createElement(Avatar, {
	userid : "claudiopro"
});

Nested transform

JSX also allows specifying children using XML syntax:


var Avatar = require('./Avatar'),
       Nav = require('./Nav');

// Input (JSX):
var app = <Nav><Avatar userid="claudiopro"/></Nav>;


// Output (JS):
var app = React.createElement(Nav,
	null,
	React.createElement(Avatar, {
		userid : "claudiopro"
	})
);
# Rendering on screen ```js var HelloMessage = React.createClass({ render: function() { return <div className="message">Hello {this.props.name}</div>; } }); ReactDOM.render(<HelloMessage name="Dublin" />, document.querySelector('#mount-point')); ```
```js var HelloMessage = React.createClass({ render: function() { return React.createElement( "div", { className: "message" }, "Hello ", this.props.name ); } }); ReactDOM.render(React.createElement(HelloMessage, { name: "Dublin" }), document.querySelector('#mount-point')); ```
# Tools - [Babel](https://babeljs.io) - Browserify - Webpack
# Babel - A JavaScript compiler. - Built out of composable plugins to create your own transformation pipeline. - Ships with built-in support for JSX.
# Babel plugins - Syntax parsers for new or experimental syntax e.g. Object rest/spread operators - Transforms e.g. ES6 modules to RequireJS or AMD - Miscellaneous e.g. undeclared variable check
# Babel presets Groups of plugins for common build profiles e.g. - React - ES6 - Experimental stage 0, 1, 2, 3
# Babel config file - A `.babelrc` file at top level in your project - Supports all flags accepted by the `babel` binary
# Getting started - Install Babel and React preset to your project ```bash npm install babel -g npm install babel-preset-react --save-dev ``` - Configure `.babelrc` ```js { "presets": ["react"] } ```
# Babel transforms - In browser transform - Build - Self-service REPL
# In browser transform - _Not for production use!_ - Use `browser.js` version that can run in a browser - Available as `babel-browser` npm module, or from a CDN - Transforms and runs scripts with type `text/ecmascript-6` and `text/babel`: ```html <script src="node_modules/babel-browser/browser.js"></script> <script type="text/babel"> class Test { test() { return "test"; } } new Test().test(); // "test" </script> ```
# Build: CLI - Install the CLI tool ```bash npm install -g babel-cli ``` - Compile a file `script.js` and write to `script-compiled.js` ```bash babel script.js --out-file script-compiled.js ``` - Watch an entire directory `src` for changes, and write to `dist` ```bash babel src --watch --out-dir dist ```
